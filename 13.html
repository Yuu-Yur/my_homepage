변수 알아보기

변수는 값이 달라지는 데이터, 상수는 값이 바뀌지 않는 데이터

변수 선언에는 규칙이 있다

영문자, _ , 숫자를 사용하고 첫 글자는 영문자, _, $를 사용하며 띄어쓰기나 기호는 불가능하다

주로 중간에 단어가 바뀌면 대문자를 쓰는 camelCase를 사용한다

  • var 변수명; 으로 선언하고 변수명 = 값; 으로 값을 할당한다 var 변수명, 변수명, 변수명; 으로 여러 변수를 한번에 선언 가능
  • var 변수명 = 값; 으로 한번에 선언과 할당을 할 수 있다
    • 자료형 이해하기

    자료형이란 컴퓨터가 처리하는 자료의 형태를 뜻한다

    숫자형, 문자열, 논리형, 배열, 객체, undefined, null 등이 있다

    • 숫자형은 정수와 실수로 나뉜다
    • 문자열은 ''나 ""로 묶은 데이터, 논리형은 true나 false의 둘 밖에 없다
    • undefined는 선언만하고 할당되지 않은 상태, null은 데이터 값이 유효하지 않은 상태로 예를들면 자료형과 값이 일치하지 않는 경우가 있다
    • 배열은 하나의 변수에 여러 값을 저장하는 형태
    • var 배열명 = [값,값,값]; 으로 선언하고 할당한다
    • 배열명[0]부터 시작해서 순서대로 번호가 매겨진다
    연산자 알아보기

    연산자에는 산술 연산자, 할당 연산자, 비교 연산자, 논리 연산자가 있다

    산술 연산자는 계산에 사용하는 연산자

    할당 연산자는 대입연산자라고도 불리며 오른쪽의 값을 왼쪽에 할당하는 연산자이다

    비교 연산자는 피연산자 2개의 값을 비교해서 true나 false를 반환하는 연산자

    논리 연산자는 true와 false가 피연산자일때 or, and, not을 거쳐 true나 false를 반환한다

    • 산술 연산자 : + - * / % ++ --
    • 할당 연산자 : = += -= *= /= %= 왼쪽 연산 오른쪽을 왼쪽에 대입
    • 비교 연산자 : == === != !== <> <=>=
    • 논리 연산자 : || && !
    조건문 알아보기

    조건문은 조건이 참일 경우 특정 명령을 실행한다

    if, if ~ else, switch 등이 있고 조건은 true나 false로만 구성된다

    • if (조건) { 명령 } 으로 구성된다
    • if (조건) {명령} else if (조건) {명령} else {명령}으로 여러 경우를 상정할 수 있다
    • 이렇게 상정할 경우가 많아질 경우 switch문을 사용할 수 있다
    • ex swiitch (조건) {case 값1 : 명령1 break; case 값2 : 명령2 break; case 값3 : 명령3 break; default : 명령4;}
    • 논리 연산자에서 중요한점 중 하나는 or의 경우 앞이 true면 뒤는 고려하지 않아도 true고 and의 경우 앞이 false면 뒤는 고려하지 않아도 false다
    • 이 경우 숏 서킷으로 뒤의 연산은 아예 실행되지 않는다 따라서 비교적 쉬운 연산을 앞에 두는것이 좋다
    반복문 알아보기

    특정 조건을 만족하지 않을때까지 명령을 반복하는 문이다

    주로 for문과 while문을 쓰며 카운터 변수를 선언하는것이 특징이다

    • ex for(var i=0; i lessthan n; i++){명령}
    • 중첩을 사용해서 n차원적 반복을 가능하게 한다
    • ex for(var i=0; i lessthan n; i++){명령; for(var j = 0; j less than m; j++){명령}}
    • while 문은 카운터 변수 선언과 초기화를 문 밖에서 하고 카운터 증가를 명령에 포함 시켜 발동한다
    • ex var i = 0; while(i lessthan n) {명령; i++}
    • 위의 조건문은 전부 조건이 true라면 명령을 실행하지만 명령을 실행한 후 조건을 체크하는 것이 do while문이다
    • ex do {명령} while(조건)
    • 조건문을 컨트롤 하기 위한 방법으로 break문과 continue문이 있다
    • break문은 종료 조건 전에 반복문을 빠져 나올 때, continue문은 특정 조건에서 명령을 스킵할 때 사용

    함수 알아보기

    함수란 명령을 묶어 놓은것이다

    함수를 호출하여 사용할 수 있어 유지보수에 이점이 있다

    function 함수명 {명령} 으로 선언 가능하다

    var를 사용한 변수의 특징

    var은 변수 선언 예약어 중의 하나이다

    가장 간단한 변수 선언어로 호이스팅, 재선언, 재할당 문제때문에 최근엔 잘 쓰이지 않는다

    • 변수는 기본적으로 선언 - 사용의 순서지만 순서가 역전되어도 일단은 작동하는것으로 버그의 근원이다
    • 재선언, 재할당이 가능하기에 의도하지 않은 상황에서 의도하지 않은 변수를 사용할 가능성이 있다
    • 전역변수는 스크립트 전체에서 사용하며 함수 밖에서 선언하거나 함수내에서 선언예약어 없이 선언하는 경우에 전역함수가 된다
    • 지역변수는 함수 내에서 선언된 변수로 함수 내에서만 작동한다
    let와 const의 등장

    위와 같은 var 선언의 피해를 막고자 let과 const 선언이 등장했다

    • let 선언 : 블록 변수로 선언이 가능하며 재선언이 불가하고 호이스팅 문제가 발생하지 않는다
    • 블록 변수 : 블록 (){} 안에서만 사용이 가능한 변수이다
    • const 선언 : 재선언,재할당이 전부 불가능하여 상수를 선언할 때 사용한다
    • 변수를 선언할 때는 가능한 전역함수를 줄이고 함수의 시작부에서 선언한다
    재사용할 수 있는 함수 만들기

    기본 함수 외에 필요한 함수를 만들어 재사용 할 수 있다

    • 매개 변수,return : 함수 밖에서 인수로 값을 받고 return으로 함수결과값을 출력할 수 있다
    함수 표현식

    함수를 선언하고 할당할 때 표현하는 방법들이다

    • 익명 함수 : 이름이 없는 함수로서 function()로 표현하며 대부분 일회용 함수로 사용한다
    • 즉시 실행 함수 : 말 그대로 호출 없이 즉시 실행되며 식 형태로 표현된다
    • ex (function(매개변수) {명령}(인수));
    • 화살표 함수 : 익명 함수에서만 사용할 수 있는 함수표현의 일종이다 (매개변수) => 명령;의 형태이다
    • ex const 변수명 = (매개변수) => 명령;
    이벤트와 이벤트 처리기

    이벤트란 웹에서 사용자의 동작을 의미하며 이벤트 처리기란 이러한 이벤트가 발생했을 때 작동하는 함수를 뜻한다

    이벤트엔 마우스, 키보드, 문서 로딩, 폼 등에서 발생하는 이벤트가 있다

    • 마우스 이벤트엔 click, dbclick, mouse(over, out, down, up, move, enter, leave)
    • 키보드 이벤트는 key(down, up, press)
    • 문서 로딩 이벤트엔 abort error load resize scroll unload
    • 폼 이벤트엔 reset focus change submit 등이 있다
    • 이러한 이벤트들에 on을 붙여 html에서 태그 on이벤트 = "함수"로 이벤트 처리기를 사용할 수 있다
    • ex Ch.11에서 마우스를 올려 트랜스폼시키는것에 mouseover과 mouseout의 이벤트를 on을 붙여 이벤트 처리기로 작동시켰다
    DOM을 이용한 이벤트 처리기

    DOM이란? 문서 객체 모델로 html의 각 요소들을 객체로 나타내고 접근하는 방법이다

    이 방법을 사용해 html문서에 on이벤트로 처리하지 않아도 js파일에서 id나 class로 원하는 태그를 지정하여 이벤트 처리가 가능하다

    • 먼저 queryselector()나 getelementbyid,class,tagname()으로 요소를 선택한다 보통 최상위는 document이고 .으로 하위로 내려간다
    • 자체로서 html의 특정,다수의 객체를 의미하지만 유지보수문제로 주로 변수에 넣어 사용한다 ex var side = document.queryselctor('#side');
    • 주의점으로 쿼리셀렉터에선 id나 클래스의 경우 #과 .을 붙여야 하지만 getelementby에선 함수에서 id, class, tag를 지정하기때문에 그대로 쓰면된다
    • 이렇게 요소를 불러왔다면 그 요소에 필요한 처리를 가한다
    • ex1 side.onclick = 함수; 이 경우 함수뒤에 ()는 필요없다 ()는 어디까지나 매개변수를 나타내기 위함이지 함수의 이름이 아니기 때문이다
    • ex2 document.queryselector('#side').onclick = 함수; 변수를 사용하지 않고 직접 지정했다 한줄일 경우엔 문제가 없지만 3줄, 10줄, 100줄로 늘면 점점 힘들어질 것이다
    • ex3 side.onclick = function() {명령}; 함수를 따로 선언하지 않고 익명함수를 사용하여 일회용으로 지정했다 단 한번만 사용할 함수라면 따로 변수를 지정해 선언하는것보다 경제적이다

    객체 알아보기

    객체란 프로그램에서 인식할 수 있는 모든 대상을 뜻하며 데이터를 저장하고 처리하는 기본 단위이다

    • 기본 객체(DOM, 브라우저객체, 내장객체)도 자주 쓰지만 커스텀 객체를 사용할 수 있는것이 큰 장점이다
    • 커스텀 객체를 만들 땐 new 객체명 으로 만든다
    • 객체는 크게 property와 method로 만들어진다 property는 속성, method는 객체의 동작을 의미한다
    • ex Date()라는 기본 객체엔 toLocaleString()라는 method가 내장되어있다
    자바스크립트와 내장 객체

    자주 쓰는 내장 객체에는 array, date, math 가 있다

    • Array 객체 배열객체로 여러 변수를 한번에 관리할 수 있다 주로 반복문과 조합하여 쓸 수 있다
    • ex1 var 변수명 : new array(n); 으로 크기를 지정할수도 지정하지 않을 수도 있다
    • ex2 var 변수명 = [값1,값2,값3,값4]; 의 배열선언을 객체를 사용하면 var 변수명 = array(값1,값2,값3,값4);로 사용한다
    • 배열의 크기는 변수명.length 이며 쉽게말해 값의 개수이다 주로 반복문에서 카운터 lessthan 배열크기로 배열 전체에 무언가를 할때 사용할 수 있다
    • array객체의 method는 많지만 주로 쓰는 것으론 concat(), join(), push(), unshift(), pop(), shift(), splice(), slice()가 있다
    • 각각 합치기, 합쳐서 문자열로, 뒤에 추가, 앞에 추가, 뒤에서 꺼내기, 앞에서 꺼내기, 중간에서 교체, 중간에서 제거의 기능이 있다
    • date 객체는 주로 날짜와 관련된 기능에 사용된다
    • ex new Date(); 로 사용한다
    • date객체의 method중 주로 사용하는 것에 getTime()이 있다
    • math객체는 계산과 관련된 기능에 사용된다
    • math 객체의 method중엔 abs(), floor(), ceil(), round(), random() 등이 주로 사용된다
    • 각각 절대값, 소수점이하 버림, 소수점이하 올림, 소수점이하 반올림, 0~1사이의 랜덤한 수 반환이다
    브라우저와 관련된 객체

    웹 브라우저에 관련된 객체로 window, document, navigator, history, location, screen등이 있다

    • window객체의 property에서 주로 쓰는것은 document, innerheight, innerwidth, location, parent
    • window에서 주로 쓰는 method는 alert(), confirm(), prompt()

    Dom 요소에 접근하기

    요소를 찾아가는것을 접근한다고 표현한다

    • getElementById(id명) method
    • getElementsByClassName(class명) method
    • getElementsByTagName(tag) method
    • class와 tag는 여럿이 있을 수 있으므로 HTMLCollection 에 저장됨
    • querySelector(선택자) method
    • querySelectorAll(선택자 or tag) method

    • innerText, innerHTML 요소의 내용을 수정하는 속성
    • 이 뿐만 아니라 속성 자체를 가져와서 수정할 수도 있다
    • getAttribute(),setAttribute() 속성명,값을 넣어 사용
